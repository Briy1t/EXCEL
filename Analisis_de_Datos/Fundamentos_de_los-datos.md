# Fundamentos de los Datos

## 1. Qué son los datos y por qué importan
Los datos están presentes en todas las actividades humanas: compras, búsquedas, desplazamientos, decisiones y hábitos. Las organizaciones dependen de ellos para identificar oportunidades, resolver problemas y fundamentar decisiones estratégicas.

- **Un dato** es una colección de hechos que puede presentarse en forma de números, texto, imágenes, vídeos o cualquier registro almacenable. Cuando se recopilan, organizan y transforman, permiten descubrir patrones y 
obtener conclusiones que no serían visibles de otra manera.

## 2. El proceso de análisis de datos
El análisis de datos sigue un ciclo estructurado que convierte información en decisiones:

- **Ask:** definir la pregunta o el problema.
- **Prepare:** recopilar, almacenar y organizar los datos.
- **Process:** limpiar, corregir y validar la información.
- **Analyze:** explorar, visualizar y detectar patrones.
- **Share:** comunicar resultados de forma clara y comprensible.
- **Act:** aplicar lo aprendido para resolver el problema.

`Existen variantes más detalladas, pero todas comparten la misma lógica: descubrir, preparar, modelar, comunicar y operacionalizar.`

## 3. Tipos de datos y calidad
- Tipos principales
  - **Numéricos:** cantidades medibles o contables.
  - **Texto:** nombres, códigos, descripciones.
  - **Fechas**: momentos específicos en el tiempo.
  - **Categóricos**: grupos o etiquetas.

## Problemas comunes de calidad
- Datos faltantes.
- Datos incorrectos.
- Formatos incoherentes.
- Duplicados.
- La calidad del dato es fundamental, ya que afecta directamente a la precisión y utilidad del análisis.

## 4. Hojas de cálculo: preparación y organización
Las hojas de cálculo son una herramienta inicial clave para organizar y explorar datos.

- Organización básica :Introducir datos en celdas.
  - Ajustar filas y columnas para mejorar la legibilidad.
  - Añadir encabezados claros.
  - Incorporar atributos adicionales cuando sea necesario.
- Ordenación y limpieza
  - Seleccionar rangos completos.
  - Ordenar por columnas específicas.
  - Verificar encabezados y formatos.
- Fórmulas esenciales :Sumas, promedios y operaciones básicas.

`Las fórmulas permiten analizar datos sin cálculos manuales. `

## 5. SQL: consultas esenciales
SQL permite trabajar con bases de datos de forma estructurada.

---
**Estructura básica de una consulta**
 - SELECT: columnas que se desean recuperar.
 - FROM: tabla de origen.
 - WHERE: condiciones de filtrado.

--- 
## Buenas prácticas en SQL
- Escribir consultas con formato claro.
- Seleccionar solo las columnas necesarias.
- Usar LIKE para buscar patrones.
- Evitar SELECT * cuando no sea necesario.
- Añadir comentarios para documentar.
- Utilizar alias para simplificar nombres largos.**Ejemplos habituales**
    - Filtrar empleados por código de puesto.
    - Excluir valores con <>.
    - Buscar nombres que comienzan por determinadas letras.

## 6. Visualización de datos
La visualización convierte análisis complejos en información comprensible.

## Pasos para planificar una visualización
- Explorar los datos y detectar patrones.
- Seleccionar la información relevante para el público.
- Elegir el tipo de gráfico adecuado.
- Ajustar la visualización hasta que comunique correctamente.

## Tipos de gráficos

- Líneas: evolución temporal.
- Mapas: relación entre ubicación y métricas.
- Donut: distribución por segmentos.
- Barras: comparaciones directas.
- Herramientas comunes
- Hojas de cálculo.
- Tableau.
- Python (Matplotlib, Seaborn, Plotly).
- R (ggplot2).

## 8. Ecosistema de datos y fuentes
El ecosistema de datos incluye sistemas, personas y procesos donde se generan y utilizan los datos.

- **Fuentes habituales**
- Datos históricos para detectar patrones.
- Datos actuales para entender comportamientos.
- Información de expertos que conocen el problema de negocio.

## 9. Datos frente a instinto
El instinto visceral es una sensación intuitiva sin explicación clara. Puede ser útil, pero también sesgado.
La toma de decisiones basada en datos se centra en la evidencia. Cuanto mejor se entienden los datos, más fácil es identificar qué información se necesita realmente.

## 12. Equidad, imparcialidad y responsabilidad
Los datos representan personas, por lo que deben tratarse con cuidado, respeto y responsabilidad.

- Considerar todos los datos disponibles
- Ignorar información que no encaja con nuestras expectativas genera conclusiones incompletas.

- Identificar factores circundantes
- El contexto influye en los datos; no deben analizarse de forma aislada.

- Incluir datos autodeclarados
- Reduce sesgos del observador y aporta información más precisa.

- Sobremuestreo para representar a todos los grupos
- Permite que minorías estén adecuadamente representadas.

- Equidad de principio a fin
- Debe aplicarse desde la recopilación hasta la presentación de resultados.
